NAME

openosint - Model Context Protocol (MCP) server and CLI for Open Source Intelligence

SYNOPSIS

openosint [-h] [-v] command [args ...]

DESCRIPTION

The openosint framework provides a standardized interface between large language models (LLMs) and real-world OSINT tools via the Model Context Protocol (MCP). It allows AI agents like Claude to autonomously discover, execute, and parse intelligence gathering tasks.

It can operate both as a daemonized MCP server over stdio and as a standalone command-line interface for manual execution.

ARCHITECTURE

The system is strictly divided into three layers:

  • Core Modules: Asynchronous Python wrappers executing external binaries (e.g., holehe).
  • MCP Server: Exposes the core modules as tool schemas adhering to the Anthropic MCP specification.
  • CLI: An argparse-driven wrapper for direct human-in-the-loop execution.

INSTALLATION

Installation is managed via pip using the PEP 621 compliant pyproject.toml configuration.

$ git clone https://github.com/tuo-username/OpenOSINT.git
$ cd OpenOSINT
$ pip install -e .

CONFIGURATION (CLAUDE)

To register the MCP server with the Claude Code CLI, execute the following command, specifying the absolute path to the wrapper script:

$ claude mcp add openosint python /path/to/OpenOSINT/openosint/mcp_server.py

EXAMPLES

Direct manual execution via CLI to verify email presence across services:

$ openosint email target@example.com --timeout 60

Agentic execution via Claude Code (after MCP registration):

$ claude
> Analyze target@example.com using the available OSINT tools.
